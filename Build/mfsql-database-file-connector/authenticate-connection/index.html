Authenticate Connection¶

After adding all the settings the Connection must be authenticated

Step

Description

Acces s menu

After completion of the setup of the general and specific settings

Right click on the Name of the connection to expose Authentication menu

Set Commo n User Authe ntica tion

The Common user authentication is used when the General Settings : Authentication is set to Common

The user must have access to the database and table/view specified in the Connector Specific Settings.

This is a required setting

Use either a windows user name and password, or SQL Server Authentication user name and password, depending on the General Settings : SQL Server Authentication setting.

Set Permi ssion s Retri ever Authe ntica tion

The Permissions retriever is used by the Connection to establish the external repository connection based on the settings in the Connector Specific Settings.

The user must have access to the database and table/view specified in the Connector Specific Settings.

This is a required setting

Use either a windows user name and password, or SQL Server Authentication user name and password, depending on the General Settings : SQL Server Authentication setting.

Set Index er User Authe ntica tion

The Indexer user is used by the Connection in the background to perform the search indexing.  The rules for search indexing is dependent on the General Settings for indexing.

The user must have access to the database and table/view specified in the Connector Specific Settings.

This is a required setting

Use either a windows user name and password, or SQL Server Authentication user name and password, depending on the General Settings : SQL Server Authentication setting.
