 Multiple Connections

Each datasource can have multiple connections allowing for a single file data table to show different hierarchies and filters

 Multiple Databases

Each Connection connects to only one database server.

Connecting multiple database servers to a single vault is possible

1

Hierarchies to organise unstructured data in M-Files

Use multiple hierarchies by referencing columns in the source view/table to show the data in drop downs in M-Files

Each connection has only one definition for a hierarchy.  Use multiple connections to  show another hierarchy for the same data

1

Promoting file to standard metadata object

The database File can be promoted to a standard M-Files Class object.  The File is retained in the database.

Promotion is based on selecting an individual item and changing the class.

1

Delete file from database

When the file is deleted from the database it will disappear from the M-Files view, unless it has been previously promoted

1

Pull through related data for the file object

Use MFSQL Connector to pull through the metadata (e.g. customer, project, etc) related to the file. This can be automated and triggered by a workflow state change.
